## 重要: リソース名の命名規則

**すべてのリソース名に `-{あなたの名前}` を付けてください。**

例: `tanaka-day5-instance`, `Day5Stack-suzuki`

共有AWSアカウントを使用するため、名前の重複を避ける必要があります。
